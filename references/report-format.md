# Report Format

Use this reference after the `paper-hexo-deepread` skill triggers.

## Local Paths

- Blog root: `G:\Document\myBlog`
- Posts: `G:\Document\myBlog\source\_posts`
- Images: `G:\Document\myBlog\source\img`
- Style example: `G:\Document\myBlog\source\_posts\DVD.md`
- Live site: `https://zjj-low-key.github.io/`

## Frontmatter

Keep the Hexo frontmatter shape compatible with `DVD.md`.

```yaml
---
title: 'PAPERNAME: Full Paper Title'
date: YYYY-MM-DD HH:mm:ss
tags: [tag1,tag2,tag3]
categories: category
cover: /img/PAPERNAME-0.png
copyright: false
---
```

Rules:

- Preserve the `date` generated by `hexo new` unless it is clearly wrong.
- Use a quoted title when it contains a colon.
- Choose tags and category by inspecting existing posts first.
- Choose the cover image by content first: prefer the paper's teaser image when available; otherwise use the main pipeline/architecture image. Save or rename that chosen cover as `/img/papername-0.png`, and set `cover` to `/img/papername-0.png`. The cover must be a clean visual-only crop: do not include the figure caption, surrounding body text, page numbers, or other PDF layout elements in `papername-0.png`.
- Keep `copyright: false` unless the user asks otherwise.

## Metadata Block

Place this block immediately after frontmatter:

```markdown
**作者：**Author A, Author B, Author C

**单位：**Institution A, Institution B

**会议：**Venue or source, Year

**链接：**[URL](URL)
```

Use `会议` for conferences, journals, arXiv, OpenReview, or other source labels. Examples: `CVPR 2026`, `ICLR 2025`, `2026 Arxiv`, `OpenReview 2025`.

## Required Sections

Use these headings exactly:

```markdown
## 研究动机

## 核心方法

## 数据集

## 算力

## 实验结果

## 优势与不足
### 优势
### 不足

## 记忆点
```

Section expectations:

- `研究动机`: Explain the gap, why previous methods are insufficient, and the paper's central question.
- `核心方法`: Explain the pipeline and key modules. Insert architecture and method figures near the related text.
- `数据集`: Separate training and evaluation datasets when possible.
- `算力`: Report GPUs, training time, batch size, resolution, or state `论文未明确说明` if absent.
- `实验结果`: Prefer paper figure/table screenshots. Add short text only when needed to explain what the result proves.
- `优势与不足`: Distinguish paper-claimed strengths from your own critical reading. Avoid unsupported criticism.
- `记忆点`: Capture 2-5 memorable ideas, design tricks, or reusable insights.

## Image Rules

Use PNG files in `G:\Document\myBlog\source\img`.

Filename pattern:

```text
papername-0.png
papername-1.png
papername-2.png
```

Markdown syntax:

```markdown
![papername-1](/img/papername-1.png 'papername-1')
```

Requirements:

- Choose the cover image by content first: teaser first, then pipeline/architecture if no teaser exists. Save or rename the chosen cover as `papername-0.png`.
- `papername-0.png` is a special case: keep only the visual content of the selected teaser/pipeline image. Do not include its figure caption or any surrounding prose, headers, footers, or page canvas.
- Keep alt text, file stem, and title text identical.
- Extract all main-paper figures and tables from the PDF into an inventory before writing. Track figure/table id, page number, caption, and target report section.
- Prefer screenshots/crops that keep axis labels, legends, table headers, row labels, subfigure labels, and captions readable.
- Each PNG should contain one complete, standalone figure or one complete, standalone table. Do not combine unrelated figures/tables unless the paper itself treats them as one composite.
- Do not include unrelated neighboring content such as abstract text, body paragraphs, references, page numbers, another figure/table, or partial text from the next column.
- Split adjacent figure/table content into separate PNGs. Stitch multi-column or multi-page visuals only when needed to keep a single figure/table complete.
- Create a contact sheet or visual preview of all extracted PNGs and recrop anything that is truncated, mixed with surrounding text, or visually ambiguous.
- If a formula is important and fragile in Markdown, use the paper screenshot instead of manually retyping it.

## Final Review Checklist

Before publishing, verify:

- The title, authors, affiliations, venue/source, and link match the paper.
- Tags and category follow existing blog taxonomy.
- Every required section is present and non-empty.
- Every figure reference points to an existing PNG in `source\img`.
- Every main-paper figure/table is either inserted near the section that discusses it or intentionally omitted with a clear reason in the working notes/final summary.
- Every inserted PNG is complete, independent, readable, and free of unrelated surrounding PDF content.
- Cover is `/img/papername-0.png`, and `papername-0.png` is the selected teaser or pipeline/architecture image without any figure caption or surrounding PDF prose.
- Markdown math blocks use balanced `$$` delimiters.
- Claims about datasets, compute, and results are supported by the paper.
- The report is written in natural Chinese and does not contain encoding artifacts.
- `hexo g` succeeds locally before `hexo d`.
